Detailed Description

Represents points in simulation time. Is used to indicate points in simulation time at which the state of the model changes. Each point in simulation time is represented by an individual object of this class and offers its own methods for arithmetic operations.

TimeInstant desmoj.core.simulator.TimeInstant.getBeginOf ( TimeUnit  smallestUnit)

Determines that last instant prior to this instant at which a new period of the unit provided "begins", which means smaller units are zero.
Examples: Assume this TimeInstant is 28.12.2011 11:23:45:678. <br> Calling getBeginOf(TimeUnit.TimeUnit.DAYS) yields 28.12.2011 00:00:00:000 (begin of current hour).<br> Calling getBeginOf(TimeUnit.TimeUnit.MINUTES) yields 28.12.2011 11:23:00:000 (begin of current day).<br> Note that days are interpreted with respect to this instant's preferred time zone.

Parameters
smallestUnitTimeUnit : the unit to begin (i.e. smaller units set to zero); must be TimeUnit.DAYS, TimeUnit.HOURS, TimeUnit.MINUTES, TimeUnit.SECONDS or TimeUnit.MILLISECONDS.
Returns
TimeInstant: The TimeInstant at which the day/hour/minute/second/millisecond of the this TimeInstant has begun.

Sets the preferred time zone for printing this time instant. Note that this time zone does not affect the internal representation of this TimeInstant (stored as multiple of Experiment's epsilon since the start of the epoch, i.e. January 1, 1970, 00:00:00 GMT); the preferred TimeZone just servers TimeFormatters generating a convenient output.
