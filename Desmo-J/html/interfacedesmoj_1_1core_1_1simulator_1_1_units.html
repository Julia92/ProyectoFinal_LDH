Detailed Description

Deprecated:
Prefer usage of Java standard classes, e.g. java.util.concurrent.TimeUnit for time units.

The interface Units provides constants for representing units for time, length and speed. Additionally, textual representations for these units and conversion factors are given.

Any class implementing this interface gains direct access to all defined constants. It is recommended to use this interface in every class that has to represent values with units.

final double [] desmoj.core.simulator.Units.unitFactors
static
Initial value:
= { 1, 1000, 60000, 3600000, 1,
100, 1000, 1000000, 1, 1000 / 3600.0 }

Conversion factors to convert between units. To get the appropriate unit factor use the unit constant as an index. For example: Units.unitFactors[Units.MIN] gives 60000.

To convert a value from an old unit to a new unit multiply the value with the factor for the old unit and divide by the factor for the new unit. Example: value * unitFactors[oldUnit] / unitFactors[newUnit]

Note that these factors of course only make sense when used for units of the same phenomenon, so you should never attempt to convert a time unit into a length unit, for example.
