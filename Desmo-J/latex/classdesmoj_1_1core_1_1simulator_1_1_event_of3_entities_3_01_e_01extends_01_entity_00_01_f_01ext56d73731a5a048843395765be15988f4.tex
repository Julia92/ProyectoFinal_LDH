\subsection{Detailed Description}
Provides the class for user defined events to change {\bfseries three} entities' internal states. The state of a discrete model is changed by events that occur at distinct points of simulation time.

For events changing the state of {\bfseries one} or {\bfseries two} entities, refer to {\ttfamily Event} and {\ttfamily Event\-Of2\-Entities}. Events not associated to a specific entity are based on {\ttfamily \doxyref{External\-Event}{p.}{classdesmoj_1_1core_1_1simulator_1_1_external_event}}. 

For type safety it is recommended to generically assign the entity types an {\ttfamily Event\-Of3\-Entities} operates on by using the generic type {\ttfamily Event\-Of3\-Entities$<$E,F,G$>$} where {\ttfamily E}, {\ttfamily F} and {\ttfamily G} are derived from {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}}. 

All event object should be used only once. Implement the changes of state for the specific entities associated with this event by overriding the abstract method {\ttfamily \doxyref{event\-Routine(\-E who1, F who2, G who3)}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_of3_entities_3_01_e_01extends_01_entity_00_01_f_01ext56d73731a5a048843395765be15988f4_a5439a934ac6642f35f022524143be731}}.

Should you decide to derive your event from the raw type {\ttfamily Event\-Of3\-Entities} (which is not recommended), please take extra care in checking the given \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity} parameters to your special event\-Routine since any subtype of \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity} will be accepted! If your model uses several different entity types, chances are that while developing the model, wrong entity types might be passed.
