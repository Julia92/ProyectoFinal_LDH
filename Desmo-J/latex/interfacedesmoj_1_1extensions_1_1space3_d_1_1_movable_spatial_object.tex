\subsection{Detailed Description}
This interface extends the \doxyref{Spatial\-Object}{p.}{interfacedesmoj_1_1extensions_1_1space3_d_1_1_spatial_object} interface by moving animation methods. The class which implements this is meant to be a movable object in the simulation. The methods cover the setting and getting spatial transformation data.

A \doxyref{Movable\-Spatial\-Object}{p.}{interfacedesmoj_1_1extensions_1_1space3_d_1_1_movable_spatial_object} (M\-S\-O) can contain other Movable\-Spatial\-Objects as components to simulate the movable extensions/parts of an object. The other way around, a M\-S\-O can also be the child-\/\-M\-S\-O of other M\-S\-Os. In this case, the Attach and Detach methods will be deactivated. All the getting methods will return the local attributes without considering the parent M\-S\-O. The position and the orientation are represented in the parent-\/\-M\-S\-O's local coordinates. For the move methods only the move(movement, focus\-On\-Track) will be remained usable. To declare a M\-S\-O as a child-\/\-M\-S\-O, the parent M\-S\-O must be give in the constructor. This is the only way to declare it and it can't be modified afterwards.
