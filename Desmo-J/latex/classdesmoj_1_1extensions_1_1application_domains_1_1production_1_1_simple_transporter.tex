\subsection{Detailed Description}
A \doxyref{Simple\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_simple_transporter} is a simple transporter (vehicle) associated to a {\ttfamily \doxyref{Transport\-Junction}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_transport_junction}}. There it waits for goods to transport them. The goods are transported for a certain time and than released. The \doxyref{Simple\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_simple_transporter} returns to his {\ttfamily Transpsort\-Junction} where it waits again for some other goods to transport. The time it takes the transporter to return to its {\ttfamily \doxyref{Transport\-Junction}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_transport_junction}} must be specified in the parameter {\ttfamily return\-Time}. It must be some kind of {\ttfamily desmoj.\-dist.\-Real\-Dist$\ast$ } random number stream. The \doxyref{Simple\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_simple_transporter} has a certain capacity (maximum number of goods which can be carried around at once) and a minimum load (minimum number of goods which will be carried). The minimum load is one (1) unless something different is specified. The Simple\-Transporters life\-Cycle is kept simple, as it only waits in a {\ttfamily \doxyref{Transport\-Junction}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_transport_junction}} for goods to be transported. Than, after having transported the goods it returns to its {\ttfamily \doxyref{Transport\-Junction}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_transport_junction}}. The user can overwrite the method {\ttfamily \doxyref{life\-Cycle()}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1production_1_1_simple_transporter_afffbfb4ef7182827da6e8750b270d68d}} in a subclass in order to implement a different behavior.
