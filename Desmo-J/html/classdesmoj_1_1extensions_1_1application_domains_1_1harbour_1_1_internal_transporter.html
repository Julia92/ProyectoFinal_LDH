Detailed Description

A InternalTransporter represents any kind of vehicle (AGV, VC, chassis) which carries goods (containers) around in a container terminal. It has a certain loaded and empty speed when he drives in a terminal and is loaded or empty. He has also a certain transporter control which he gets his Job to do by. InternalTransporter is derived from Transporter. Because of that he has a certain capacity (maximum number of goods which can be carried around at once) and a minimum load (minimum number of goods which will be carried) that one is. Its lifeCycle() is already implemented and spezifies the behavior of the InternalTransporter. But its methods importJobCycle(Job j) and ExportJobCycle(Job j) must be ipmlemented by the user in order to specify the behavior of the InternalTransporter if he does an emport or export job.

Constructs an Internal Transporter which will carry around containers (goods) in a container terminal. Implement its importJobCycle(Job j) method and exportJobCycle(Job j) to specify its behavior when he gets an ex- or import job. An Internal Transporter has like a Transporter a capacity (maximum number of goods which can be transported) and a mimimum load ( a minimum number of goods it will carry). Both must not be zero or negative. The capacity default value is one. The minimum load has always value one.
