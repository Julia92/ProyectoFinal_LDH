\subsection{Detailed Description}
The model is supposed to carry references to all static modelcomponents associated to a model. These are modelcomponents like distributions and statistic counters. Note that placing all essential components inside the model class allows you to use object orientation to reuse this model to implement different versions of it using subclassing for changes necessary for using other modelling paradigms. This way models can be easily reimplemented following the process-\/, event-\/, transaction-\/ or activity-\/ oriented paradigm. Note that transaction-\/ and activity-\/oriented modelling requires the higher process synchronization mechanisms (Res, Bin, Wait\-Queue, Cond\-Queue) by Soenke Claassen. 

The modelcomponents belonging to a model have to be instantiated in the {\ttfamily \doxyref{init()}{p.}{classdesmoj_1_1core_1_1simulator_1_1_model_a87b82ee5c9e344c34745d4844ca72884}} method. They must not be instantiated in the model's constructor since most components need access to a valid experiment object. This can only be accessed after a model has been connected to an experiment using method {\ttfamily \doxyref{connect\-To\-Experiment(\-Experiment exp)}{p.}{classdesmoj_1_1core_1_1simulator_1_1_model_a1f4e1d7a9f14a7e68f43a79c88307ffc}} which implicitly calls the {\ttfamily \doxyref{init()}{p.}{classdesmoj_1_1core_1_1simulator_1_1_model_a87b82ee5c9e344c34745d4844ca72884}} method after receiving the valid reference to an experiment.

Implement this method to do initialization work for your model. All Model\-Components associated with this model are supposed to be instantiated in this method. 

{\itshape Caution! Do not use the \doxyref{Model}{p.}{classdesmoj_1_1core_1_1simulator_1_1_model}'s constructor method to instantiate your modelcomponents!} \par
 This prevents the modelcomponents to get contact to the associated \doxyref{Experiment}{p.}{classdesmoj_1_1core_1_1simulator_1_1_experiment} since a \doxyref{Model}{p.}{classdesmoj_1_1core_1_1simulator_1_1_model} can only be connected to an \doxyref{Experiment}{p.}{classdesmoj_1_1core_1_1simulator_1_1_experiment} after itself has been instantiated correctly. Thus it is necessary to put all initialization and instantiation work into this method. Do not use this method to do the initial schedules needed for the scheduler to start with. Use method {\ttfamily \doxyref{do\-Initial\-Schedules()}{p.}{classdesmoj_1_1core_1_1simulator_1_1_model_addaf071481a772a04520b2996e545a36}} instead. If there are no events scheduled in the event-\/list, the scheduler will stop the simulation immediately because there is nothing for him to do. If you are using submodels, take care that each of the submodels has its own {\ttfamily init} method properly set up. 

Implemented in {\bf desmoj.\-extensions.\-visualization2d.\-animation.\-core.\-simulator.\-Model\-Animation} \doxyref{}{p.}{classdesmoj_1_1extensions_1_1visualization2d_1_1animation_1_1core_1_1simulator_1_1_model_animation_af4b388117c85f6a60b3f95d52cf4e3ac}.
