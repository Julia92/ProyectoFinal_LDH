\subsection{Detailed Description}
Event notes contain any information needed for any type of {\ttfamily \doxyref{Schedulable}{p.}{classdesmoj_1_1core_1_1simulator_1_1_schedulable}}, event or process. Since {\ttfamily \doxyref{Event\-Note}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_note}} are tightly coupled with the {\ttfamily \doxyref{Event\-List}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_list}}, a change of the {\ttfamily \doxyref{Event\-List}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_list}} might alo force the {\ttfamily \doxyref{Event\-Note}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_note}} to contain other information especially concerning the data structures used in the {\ttfamily \doxyref{Event\-List}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_list}} as well. To adopt the {\ttfamily Event notes} extend them to carry the specific data and methods you need for a possible change of the {\ttfamily \doxyref{Event\-List}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_list}}. The implementation here includes data and methods for event information only, since the default implementation of class {\ttfamily Event\-Vector} needs no special information stored inside the {\ttfamily Event notes}.
