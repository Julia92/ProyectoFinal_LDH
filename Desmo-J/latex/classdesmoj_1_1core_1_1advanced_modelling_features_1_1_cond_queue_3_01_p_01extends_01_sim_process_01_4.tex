\subsection{Detailed Description}
In a Cond\-Queue processes are waiting for a specific condition to become true. Process synchronisation happens when processes are waiting in a queue for a specific condition to become true. Each process which uses {\ttfamily wait\-Until(condition)} and does not find this condition to be true, is inserted in a waiting-\/queue automatically. Whenever something happens which might influence the condition to become {\ttfamily true}, {\ttfamily \doxyref{signal()}{p.}{classdesmoj_1_1core_1_1advanced_modelling_features_1_1_cond_queue_3_01_p_01extends_01_sim_process_01_4_a2268d80aaaed75d5737c38b7ab9948aa}} should be used to check if the first entity (or, if the attribute {\ttfamily check\-All} is set to true, all entities) in the waiting-\/queue finds the desired condition now and therefore can continue. The designer of the model is responsible that this check takes place! He also has to implement the condition, this can easily be done by deriving it from the interface {\ttfamily Condition}. The flag, if only the first or all entities in the queue are checking their conditions again, can be checked and changed using {\ttfamily \doxyref{get\-Check\-All()}{p.}{classdesmoj_1_1core_1_1advanced_modelling_features_1_1_cond_queue_3_01_p_01extends_01_sim_process_01_4_ae00e2847b00d3c9335db06feaa61ab37}} and {\ttfamily \doxyref{set\-Check\-All()}{p.}{classdesmoj_1_1core_1_1advanced_modelling_features_1_1_cond_queue_3_01_p_01extends_01_sim_process_01_4_ace4583513d82fae4e1b744cf6095e677}}. The first sort criteria of the queue is always highest priorities first, the second queueing discipline of the underlying queue and the capacity limit can be determined by the user (default is F\-I\-F\-O and unlimited capacity). Cond\-Queue is derived from Queue\-Based, which provides all the statistical functionality for a queue.

Lets the current process wait in the Cond\-Queue until a certain condition, given as a parameter, has become true. When the process finds its condition to have become true or {\ttfamily check\-All} is set to {\ttfamily true} the next process in the queue will be activated, too. If the capacity limit of the queue is reached, the process will not be enqueued and {\ttfamily false} returned.
