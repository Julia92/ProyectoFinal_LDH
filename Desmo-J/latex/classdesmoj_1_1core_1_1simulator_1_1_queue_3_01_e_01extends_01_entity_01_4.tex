\subsection{Detailed Description}
Queue provides models with a ready-\/to-\/use modelling element to enqueue entities in. The sort order of the queue is determined first by the priorities of the enqueued entities and second by the given sort order. The default sort order is F\-I\-F\-O (first in, first out) but others like L\-I\-F\-O (last in, first out) can be chosen, too. See the constants in class {\ttfamily \doxyref{Queue\-Based}{p.}{classdesmoj_1_1core_1_1simulator_1_1_queue_based}} and the derived classes from {\ttfamily Queue\-List}. The capacity of the Queue, that is the maximum number of entities enqueued, can be chosen, too. 

For queueing {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}}es the usage of class {\ttfamily Process\-Queue} is recommended.

Constructs a simple priority based waiting-\/queue for entities, the kind of queue sort order and the capacity of the queue can be chosen. Note that since Sim\-Processes are derived from Entities, they can be queued inside this queue, too. 

Queues can be sorted as follows\-:
\begin{DoxyItemize}
\item {\ttfamily \doxyref{Queue\-Based.\-F\-I\-F\-O}{p.}{classdesmoj_1_1core_1_1simulator_1_1_queue_based_a20088d8d7196b2688cbea9d313b13098}} sorts entities by their priority, highest priority first. Entities with the same priority are enqueued based on "first in, first out". 
\item {\ttfamily \doxyref{Queue\-Based.\-L\-I\-F\-O}{p.}{classdesmoj_1_1core_1_1simulator_1_1_queue_based_a3b5be66f9f00e1689906f32c952fc4e2}} also sorts entities by their priority, highest priority first. However, entities with the same priority are enqueued based on "last in, first out". 
\item {\ttfamily Queue\-Based.\-Random} assigns a random position to each entity entering the queue, disregarding priority. 
\end{DoxyItemize}

The usage of the generic version {\ttfamily Queue$<$Type$>$} where {\ttfamily Type} is derived from {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}} is recommended for type safety. Using the raw type {\ttfamily Queue} yields a queue in which any {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}} can be enqueued, typically requiring type casting on accessing entities enqueued.
