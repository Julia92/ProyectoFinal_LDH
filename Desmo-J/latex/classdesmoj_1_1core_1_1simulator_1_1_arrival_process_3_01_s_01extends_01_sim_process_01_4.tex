\subsection{Detailed Description}
An {\ttfamily Arrival\-Process} is some kind of source for {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} es. It makes them arrive from the 'outside world' in the system of interest. So it can be viewed as the border of the system (model). The user has to specify at which rate (frequency) {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} es are arriving in the system. See the Constructor for more details. The user has to derive from this class to make his own Arrival\-Processes for the arriving processes he needs in his simulated system. He has to implement the abstract method {\ttfamily \doxyref{create\-Successor()}{p.}{classdesmoj_1_1core_1_1simulator_1_1_arrival_process_3_01_s_01extends_01_sim_process_01_4_a2356daf4ef57bef4d9ccced693446159}}, which should create (instantiate) the newly arriving process. It is recommended to use one Arrival\-Process for each different kind of arriving process (with its specific arrival rate).

The {\ttfamily Arrival\-Process} is some kind of source for {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} es. So its life\-Cycle is quite simple\-: Make a new {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} object and activate it. Then wait until the next {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} object is arriving at the system. Note\-: the first {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} object is arriving when this {\ttfamily Arrival\-Process} is started! This method uses the {\ttfamily \doxyref{create\-Successor()}{p.}{classdesmoj_1_1core_1_1simulator_1_1_arrival_process_3_01_s_01extends_01_sim_process_01_4_a2356daf4ef57bef4d9ccced693446159}} method to create a new {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} object. If this method returns {\ttfamily null} the life cycle of this {\ttfamily Arrival\-Process} will stop.
