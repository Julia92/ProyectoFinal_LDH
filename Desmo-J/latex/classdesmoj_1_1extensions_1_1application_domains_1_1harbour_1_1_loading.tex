\subsection{Detailed Description}
\doxyref{Loading}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_loading} is the object representing the process cooperation between a {\ttfamily \doxyref{Internal\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter}} or {\ttfamily \doxyref{Crane}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_crane}} process and the \doxyref{Truck}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_truck} or an Internal Transporter he is loading. It is intended that this \doxyref{Loading}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_loading} is used with the {\ttfamily Wait\-Queue} construct, where a {\ttfamily Inernal\-Transporter} or {\ttfamily \doxyref{Crane}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_crane}}$>$( as a master process) is waiting for a truck or an internal transporter to load it. During the \doxyref{Loading}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_loading} the master is active and the slaves is passive. The unloading carried out together is described in the method {\ttfamily cooperation}, which can be overwritten by the user to build more complex models. Until now this method only models the time it takes to load the truck/ an internal transporter.

This method describes the loading process carried out by the master process (some kind of {\ttfamily \doxyref{Internal\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter}} or {\ttfamily \doxyref{Crane}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_crane}}). In this simple case only the time it takes to load the containers (goods) is taken into consideration. If the user building the model has to implement a more complex behavior, he has to overwrite this method in a subclass.
