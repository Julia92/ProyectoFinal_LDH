\subsection{Detailed Description}
Provides the class for user defined events to change {\bfseries a single} entity's internal state. The state of a discrete model is changed by events that occur at distinct points of simulation time.

For events changing the state of {\bfseries two} or {\bfseries three} entities, refer to {\ttfamily Event\-Of2\-Entities} and {\ttfamily Event\-Of3\-Entities}. Events not associated to a specific entity are based on {\ttfamily \doxyref{External\-Event}{p.}{classdesmoj_1_1core_1_1simulator_1_1_external_event}}. 

For type safety it is recommended to generically assign the entity type an Event operates on by using the generic type {\ttfamily Event$<$E$>$} where {\ttfamily E} is derived from {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}}. 

All event object should be used only once. Implement the changes of state for the specific entity associated with this event by overriding the abstract method {\ttfamily \doxyref{event\-Routine(\-E who)}{p.}{classdesmoj_1_1core_1_1simulator_1_1_event_3_01_e_01extends_01_entity_01_4_a1442594496eedeaecbd316898799261a}}.

Implement this abstract method to express the changes of state this event does to a single entity. This event is related to the entity it has been scheduled with. That \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity} is given note that since class {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} inherits from class {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}}, an event can also be given a process to operate on. In this case, the process is scheduled and manipulated by this event just like an entity.

Implement this abstract method to express the changes of state this event does to an entity (as {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}} is a subclass of {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}}, a process can be passed as well). 

For type safety, it is recommended to derive your events from the generic type {\ttfamily Event$<$Entity\-Operating\-On$>$} where {\ttfamily Entity\-Operating\-On} (which is derived from class {\ttfamily \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity}}) represents the entity type your event is supposed to operate on. 

Should you decide to derive your event from the raw type {\ttfamily Event} (which is not recommended), please take extra care in checking the given \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity} parameter to your special event\-Routine since any subtype of \doxyref{Entity}{p.}{classdesmoj_1_1core_1_1simulator_1_1_entity} will be accepted! If your model uses several different entity types, chances are that while developing the model, wrong entity types might be passed.
