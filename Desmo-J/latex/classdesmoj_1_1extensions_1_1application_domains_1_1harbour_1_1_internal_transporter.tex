\subsection{Detailed Description}
A \doxyref{Internal\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter} represents any kind of vehicle (A\-G\-V, V\-C, chassis) which carries goods (containers) around in a container terminal. It has a certain loaded and empty speed when he drives in a terminal and is loaded or empty. He has also a certain transporter control which he gets his {\ttfamily \doxyref{Job}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_job}} to do by. \doxyref{Internal\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter} is derived from Transporter. Because of that he has a certain capacity (maximum number of goods which can be carried around at once) and a minimum load (minimum number of goods which will be carried) that one is. Its {\ttfamily \doxyref{life\-Cycle()}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter_abe3c62357c772f13e4307dff335adb7c}} is already implemented and spezifies the behavior of the \doxyref{Internal\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter}. But its methods {\ttfamily \doxyref{import\-Job\-Cycle(\-Job j)}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter_abcf92d061120dfb76e4dce1c12ba11b5}} and {\ttfamily Export\-Job\-Cycle(\-Job j)} must be ipmlemented by the user in order to specify the behavior of the \doxyref{Internal\-Transporter}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter} if he does an emport or export job.

Constructs an Internal Transporter which will carry around containers (goods) in a container terminal. Implement its {\ttfamily \doxyref{import\-Job\-Cycle(\-Job j)}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter_abcf92d061120dfb76e4dce1c12ba11b5}} method and {\ttfamily \doxyref{export\-Job\-Cycle(\-Job j)}{p.}{classdesmoj_1_1extensions_1_1application_domains_1_1harbour_1_1_internal_transporter_a75e895f8d93d684cd28f398aab453685}} to specify its behavior when he gets an ex-\/ or import job. An Internal Transporter has like a Transporter a capacity (maximum number of goods which can be transported) and a mimimum load ( a minimum number of goods it will carry). Both must not be zero or negative. The capacity default value is one. The minimum load has always value one.
