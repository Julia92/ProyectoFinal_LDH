\subsection{Detailed Description}
Process\-Queue provides models with a ready-\/to-\/use element to enqueue {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}}es in. The sort order of the Process\-Queue is determined first by the priorities of the enqueued Sim\-Processes and second by the given sort order. The default sort order is F\-I\-F\-O (first in, first out) but others like L\-I\-F\-O (last in, first out) can be chosen, too. See the constants in class {\ttfamily \doxyref{Queue\-Based}{p.}{classdesmoj_1_1core_1_1simulator_1_1_queue_based}} and the derived classes from {\ttfamily Queue\-List}. The capacity of the Process\-Queue, that is the maximum number of Sim\-Processes enqueued, can be chosen, too. Note that in contrast to the 'plain' queue, this Process\-Queue always expects and returns objects that are derived from class {\ttfamily \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}}. When modelling using the process-\/, activity-\/, or transaction-\/oriented paradigm where Sim\-Processes are used to represent the model's entities, this Process\-Queue can be used instead of the standard Queue to reduce the amount of casts needed otherwise.

The \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process} will be stored in the Process\-Queue until method {\ttfamily \doxyref{remove(\-Sim\-Process p)}{p.}{classdesmoj_1_1core_1_1simulator_1_1_process_queue_3_01_p_01extends_01_sim_process_01_4_a8d6f5a926773a41d2ce6322136be2dda}} is called with this specific \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process}. The first \doxyref{Sim\-Process}{p.}{classdesmoj_1_1core_1_1simulator_1_1_sim_process} inside the process-\/queue will always be the one with the highest priority.
