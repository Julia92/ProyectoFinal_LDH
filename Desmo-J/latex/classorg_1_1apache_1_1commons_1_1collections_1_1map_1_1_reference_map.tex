\subsection{Detailed Description}
A {\ttfamily Map} implementation that allows mappings to be removed by the garbage collector. 

When you construct a {\ttfamily \doxyref{Reference\-Map}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1map_1_1_reference_map}}, you can specify what kind of references are used to store the map's keys and values. If non-\/hard references are used, then the garbage collector can remove mappings if a key or value becomes unreachable, or if the J\-V\-M's memory is running low. For information on how the different reference types behave, see \doxyref{Reference}{p.}{}. 

Different types of references can be specified for keys and values. The keys can be configured to be weak but the values hard, in which case this class will behave like a {\tt {\ttfamily Weak\-Hash\-Map}}. However, you can also specify hard keys and weak values, or any other combination. The default constructor uses hard keys and soft values, providing a memory-\/sensitive cache. 

This map is similar to \doxyref{Reference\-Identity\-Map}{p.}{}. It differs in that keys and values in this class are compared using {\ttfamily \doxyref{equals()}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1map_1_1_abstract_hashed_map_a51f160d58d2e1c71c78911f89b4b1b5d}}. 

This \doxyref{Map}{p.}{} implementation does {\itshape not} allow null elements. Attempting to add a null key or value to the map will raise a {\ttfamily Null\-Pointer\-Exception}. 

This implementation is not synchronized. You can use \doxyref{java.\-util.\-Collections\#synchronized\-Map}{p.}{} to provide synchronized access to a {\ttfamily \doxyref{Reference\-Map}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1map_1_1_reference_map}}. Remember that synchronization will not stop the garbage collecter removing entries. 

All the available iterators can be reset back to the start by casting to {\ttfamily \doxyref{Resettable\-Iterator}{p.}{interfaceorg_1_1apache_1_1commons_1_1collections_1_1_resettable_iterator}} and calling {\ttfamily reset()}. 

{\bfseries Note that \doxyref{Reference\-Map}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1map_1_1_reference_map} is not synchronized and is not thread-\/safe.} If you wish to use this map from multiple threads concurrently, you must use appropriate synchronization. The simplest approach is to wrap this map using \doxyref{java.\-util.\-Collections\#synchronized\-Map}{p.}{}. This class may throw exceptions when accessed by concurrent threads without synchronization. 

N\-O\-T\-E\-: As from Commons Collections 3.\-1 this map extends {\ttfamily \doxyref{Abstract\-Reference\-Map}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1map_1_1_abstract_reference_map}} (previously it extended Abstract\-Map). As a result, the implementation is now extensible and provides a {\ttfamily \doxyref{Map\-Iterator}{p.}{interfaceorg_1_1apache_1_1commons_1_1collections_1_1_map_iterator}}.
