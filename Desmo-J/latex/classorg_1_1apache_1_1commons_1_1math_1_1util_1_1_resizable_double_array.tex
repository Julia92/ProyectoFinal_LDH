\subsection{Detailed Description}
A variable length \doxyref{Double\-Array}{p.}{interfaceorg_1_1apache_1_1commons_1_1math_1_1util_1_1_double_array} implementation that automatically handles expanding and contracting its internal storage array as elements are added and removed. 

The internal storage array starts with capacity determined by the {\ttfamily initial\-Capacity} property, which can be set by the constructor. The default initial capacity is 16. Adding elements using \doxyref{add\-Element(double)}{p.}{classorg_1_1apache_1_1commons_1_1math_1_1util_1_1_resizable_double_array_a92bf109c1bd11b2b8ad312bbed29e13e} appends elements to the end of the array. When there are no open entries at the end of the internal storage array, the array is expanded. The size of the expanded array depends on the {\ttfamily expansion\-Mode} and {\ttfamily expansion\-Factor} properties. The {\ttfamily expansion\-Mode} determines whether the size of the array is multiplied by the {\ttfamily expansion\-Factor} (M\-U\-L\-T\-I\-P\-L\-I\-C\-A\-T\-I\-V\-E\-\_\-\-M\-O\-D\-E) or if the expansion is additive (A\-D\-D\-I\-T\-I\-V\-E\-\_\-\-M\-O\-D\-E -- {\ttfamily expansion\-Factor} storage locations added). The default {\ttfamily expansion\-Mode} is M\-U\-L\-T\-I\-P\-L\-I\-C\-A\-T\-I\-V\-E\-\_\-\-M\-O\-D\-E and the default {\ttfamily expansion\-Factor} is 2.\-0. 

The \doxyref{add\-Element\-Rolling(double)}{p.}{classorg_1_1apache_1_1commons_1_1math_1_1util_1_1_resizable_double_array_a503cbf4a4a334030e27a5a63d2456f4c} method adds a new element to the end of the internal storage array and adjusts the \char`\"{}usable window\char`\"{} of the internal array forward by one position (effectively making what was the second element the first, and so on). Repeated activations of this method (or activation of \doxyref{discard\-Front\-Elements(int)}{p.}{classorg_1_1apache_1_1commons_1_1math_1_1util_1_1_resizable_double_array_a7cbc39dbb4280a2e5877bd824a838c99}) will effectively orphan the storage locations at the beginning of the internal storage array. To reclaim this storage, each time one of these methods is activated, the size of the internal storage array is compared to the number of addressable elements (the {\ttfamily num\-Elements} property) and if the difference is too large, the internal array is contracted to size {\ttfamily num\-Elements + 1.} The determination of when the internal storage array is \char`\"{}too large\char`\"{} depends on the {\ttfamily expansion\-Mode} and {\ttfamily contraction\-Factor} properties. If the {\ttfamily expansion\-Mode} is {\ttfamily M\-U\-L\-T\-I\-P\-L\-I\-C\-A\-T\-I\-V\-E\-\_\-\-M\-O\-D\-E}, contraction is triggered when the ratio between storage array length and {\ttfamily num\-Elements} exceeds {\ttfamily contraction\-Factor.} If the {\ttfamily expansion\-Mode} is {\ttfamily A\-D\-D\-I\-T\-I\-V\-E\-\_\-\-M\-O\-D\-E,} the number of excess storage locations is compared to {\ttfamily contraction\-Factor.} 

To avoid cycles of expansions and contractions, the {\ttfamily expansion\-Factor} must not exceed the {\ttfamily contraction\-Factor.} Constructors and mutators for both of these properties enforce this requirement, throwing Illegal\-Argument\-Exception if it is violated.
