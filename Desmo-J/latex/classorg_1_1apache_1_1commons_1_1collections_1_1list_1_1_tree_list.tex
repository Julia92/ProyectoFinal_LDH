\subsection{Detailed Description}
A {\ttfamily List} implementation that is optimised for fast insertions and removals at any index in the list. 

This list implementation utilises a tree structure internally to ensure that all insertions and removals are O(log n). This provides much faster performance than both an {\ttfamily Array\-List} and a {\ttfamily Linked\-List} where elements are inserted and removed repeatedly from anywhere in the list. 

The following relative performance statistics are indicative of this class\-: 
\begin{DoxyPre}
             get  add  insert  iterate  remove
\doxyref{TreeList}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1list_1_1_tree_list}       3    5       1       2       1
ArrayList      1    1      40       1      40
LinkedList  5800    1     350       2     325
\end{DoxyPre}
 {\ttfamily Array\-List} is a good general purpose list implementation. It is faster than {\ttfamily \doxyref{Tree\-List}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1list_1_1_tree_list}} for most operations except inserting and removing in the middle of the list. {\ttfamily Array\-List} also uses less memory as {\ttfamily \doxyref{Tree\-List}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1list_1_1_tree_list}} uses one object per entry. 

{\ttfamily Linked\-List} is rarely a good choice of implementation. {\ttfamily \doxyref{Tree\-List}{p.}{classorg_1_1apache_1_1commons_1_1collections_1_1list_1_1_tree_list}} is almost always a good replacement for it, although it does use sligtly more memory.
