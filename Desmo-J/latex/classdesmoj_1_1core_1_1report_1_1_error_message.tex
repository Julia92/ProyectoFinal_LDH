\subsection{Detailed Description}
An \doxyref{Error\-Message}{p.}{classdesmoj_1_1core_1_1report_1_1_error_message} is used to signal invalid situations to the modeller. An \doxyref{Error\-Message}{p.}{classdesmoj_1_1core_1_1report_1_1_error_message} consists of four Strings that intend to define the kind of error that occurred and describe its reason and what a modeller can do to prevent it. Error\-Messages describe types of errors that can be fixed by the D\-E\-S\-M\-O-\/\-J or just give hints on missing or unclear parameters where certain framework assumptions are made. Error\-Messages report these to the modeller. Any type of error consists of the following attributes which have to be set by the \doxyref{Error\-Message}{p.}{classdesmoj_1_1core_1_1report_1_1_error_message}'s constructor\-: 
\begin{DoxyItemize}
\item error description \-: The operation that just failed or does not have the preconditions matched 
\item error location \-: The class and method that the error occurred in 
\item error reason \-: The probable reason why the error occured 
\item error prevention \-: A hint on how to prevent this error to occur again.  
\item error time \-: The point of simulation time that the error occurred. 
\end{DoxyItemize}Note that Error\-Messages relate to the modeller's use of the framework's methods and correct the found problems if possible by making assumptions. These corrections and their assumptions are articulated by an \doxyref{Error\-Message}{p.}{classdesmoj_1_1core_1_1report_1_1_error_message}. In contrast to {\ttfamily Exceptions} and fatal errors, an \doxyref{Error\-Message}{p.}{classdesmoj_1_1core_1_1report_1_1_error_message} does not stop the simulation run or exit the Java runtime.
